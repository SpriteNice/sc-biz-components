# 介绍

## sc-biz-components 是什么？

sc-biz-components 是一个基于 Ant Design 的企业级业务组件库，采用**源码级管理**方式。与传统的 npm 包不同，它通过 CLI 工具将组件源码直接添加到你的项目中，让你完全掌控代码。

## 为什么选择源码级管理？

### 传统方案的局限

传统的组件库通过 npm 安装，虽然方便，但存在一些问题：

- 🔒 **黑盒依赖**：组件代码在 node_modules 中，难以调试和修改
- 📦 **包体积臃肿**：即使只用一个组件，也要安装整个库
- 🔧 **定制困难**：需要通过配置或二次封装，灵活性受限
- 🔄 **版本升级风险**：升级可能带来破坏性变更

### 源码级管理的优势

- ✅ **完全可控**：源码在你的项目中，可以随时查看和修改
- ✅ **按需添加**：只添加需要的组件，保持项目轻量
- ✅ **灵活定制**：直接修改源码，满足特殊业务需求
- ✅ **无版本困扰**：代码属于你，不受上游版本影响

## 核心特性

### 🎯 源码级别管理

通过 CLI 工具将组件源码复制到项目的 `src/components` 目录，代码完全透明可控。

### 🚀 开箱即用

针对企业常见业务场景深度封装，如自定义日历、业务卡片等，开箱即用，显著提升开发效率。

### 🎨 基于 Ant Design

完全兼容 Ant Design 设计体系，可与现有 Ant Design 项目无缝集成。

### 💡 TypeScript 优先

所有组件和 Hooks 都用 TypeScript 编写，提供完整的类型定义和智能提示。

### 🛠️ 强大的 CLI

提供便捷的命令行工具：
- `sc-biz init` - 初始化配置
- `sc-biz add <component>` - 添加组件
- `sc-biz list` - 查看可用组件

### 🌈 现代化样式方案

采用 CSS Modules + Less，样式完全隔离，支持主题定制。

## 适用场景

- 🏢 **企业中后台系统**：OA、ERP、CRM 等管理系统
- 📊 **数据可视化平台**：BI 报表、监控大屏
- 🛠️ **运营管理后台**：商品管理、订单管理、用户管理
- 📱 **内部工具系统**：审批流程、权限管理

## 设计理念

我们相信：

- **代码应该是透明的**：开发者有权知道和修改使用的每一行代码
- **组件应该是灵活的**：不同项目有不同需求，组件应该易于定制
- **开发应该是高效的**：封装最佳实践，减少重复劳动
- **项目应该是轻量的**：只添加需要的，避免冗余
